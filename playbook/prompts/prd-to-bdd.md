### PRD → BDD
```text
Read:
- .ai/features/<feature>/prd.md

Generate BDD scenarios covering:
- happy path
- edge cases
- failure cases
- configuration-driven behavior
```

### Consistency check
```text
Read:
- .ai/features/<feature>/prd.md
- .ai/features/<feature>/bdd.md
- relevant tests

Check consistency between PRD, BDD and tests.
List missing or obsolete rules, scenarios and tests.
```
